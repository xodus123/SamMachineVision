## MVXTester 기본 사용법

### 개요

SamMachineVision(MVXTester)은 노드 기반 비주얼 프로그래밍 환경으로, 코드 작성 없이 이미지 처리 파이프라인을 구축할 수 있습니다. 노드를 캔버스에 배치하고 연결하여 컴퓨터 비전 워크플로우를 만듭니다.

### 시작하기

1. 좌측 노드 팔레트(Node Palette)에서 노드를 캔버스로 드래그합니다
2. 출력 포트(우측)에서 입력 포트(좌측)로 드래그하여 노드를 연결합니다
3. 우측 Properties 패널에서 노드 파라미터를 설정합니다
4. F5를 눌러 실행하거나 Auto Execute를 활성화합니다

### Dirty(더티) 상태란?

Dirty 상태는 노드가 "재실행이 필요한 상태"를 의미합니다. 노드의 속성(파라미터)이 변경되거나, 연결된 입력 데이터가 업데이트되면 해당 노드는 dirty 상태가 됩니다. dirty 상태인 노드만 다시 실행하면 되므로, 변경되지 않은 노드는 건너뛰어 실행 속도가 빨라집니다. dirty 상태는 하류(downstream) 노드로 전파됩니다. 예를 들어 Threshold 노드의 파라미터를 변경하면, 그 뒤에 연결된 Find Contours, Draw Contours 노드도 모두 dirty가 되어 함께 재실행됩니다.

### 실행 모드

#### Execute (F5) — Runtime 모드
연속 실행 모드입니다. dirty 상태(속성 변경, 입력 업데이트)인 노드만 재실행합니다. 파라미터를 조정하면 변경된 노드와 그 하류 노드만 다시 처리되므로, 인터랙티브 파라미터 튜닝에 적합합니다. 다시 F5를 누르면 Runtime을 중지합니다.

#### Force Execute (Ctrl+F5) — 단일 실행
dirty 상태와 관계없이 모든 노드를 강제로 한 번 재처리합니다. 전체 파이프라인을 처음부터 다시 실행하고 싶을 때 사용합니다.

#### Stream (F6) — 연속 스트리밍 모드
최대 FPS로 매 프레임마다 모든 노드를 재실행합니다. 라이브 카메라 피드와 실시간 처리에 사용됩니다.

#### Auto Execute
체크하면 속성이 변경될 때 자동으로 그래프를 재실행합니다 (150ms 디바운스). Runtime 모드(F5) 실행 중에만 동작합니다.

### 노드 연결 규칙

- 각 입력 포트는 하나의 연결만 허용합니다 (기존 연결은 대체됨)
- 출력 포트는 여러 입력 포트에 연결할 수 있습니다
- 연결은 타입 호환성을 강제합니다 (예: Mat → Mat, int → int)
- 순환 연결(Cycle)은 자동으로 감지되어 방지됩니다
- 데이터 흐름: 좌에서 우로, Output → Input (pull-based)

### 툴바 도구 기능

툴바에는 다음 기능 버튼이 있습니다:
- New (Ctrl+N): 새 그래프 생성
- Open (Ctrl+O): 프로젝트 파일 열기
- Save (Ctrl+S) / Save As (Ctrl+Shift+S): 프로젝트 저장
- Import Function: 함수 노드 가져오기 (.mvxp 파일을 재사용 가능한 노드로 등록)
- Python Code: 현재 그래프를 Python(OpenCV) 코드로 변환
- C# Code: 현재 그래프를 C#(OpenCvSharp) 코드로 변환
- Execute (F5): 파이프라인 실행 (dirty 노드만 재실행)
- Force Execute (Ctrl+F5): 모든 노드 강제 재실행
- Stream (F6): 연속 스트리밍 모드 (실시간 카메라용)
- Auto Execute: 속성 변경 시 자동 재실행
- HelperBot: AI 도움말 챗봇
- Help (F1): 사용자 설명서
- 테마 전환: Dark/Light 테마

#### 새 그래프 (Ctrl+N)
현재 그래프를 초기화하고 빈 캔버스를 만듭니다.

#### 프로젝트 열기 (Ctrl+O)
저장된 프로젝트 파일(.mvxp 또는 .mvx)을 열어 그래프를 불러옵니다.

#### 저장 (Ctrl+S) / 다른 이름으로 저장 (Ctrl+Shift+S)
현재 그래프를 프로젝트 파일로 저장합니다.

#### Import Function — 함수 노드 가져오기
저장된 프로젝트 파일(.mvxp 또는 .mvx)을 함수 노드로 등록하는 기능입니다. 복잡한 처리 파이프라인을 하나의 재사용 가능한 노드로 만들 수 있습니다.
- 버튼 클릭 → 파일 선택 다이얼로그에서 .mvxp/.mvx 파일 선택
- 파일명이 함수 노드의 이름이 됩니다 (예: image_preprocessing.mvxp → "image_preprocessing" 노드)
- 노드 팔레트의 Function 카테고리에 자동으로 추가됩니다
- 다른 프로젝트에서 드래그앤드롭으로 사용할 수 있습니다
- 함수 노드를 더블클릭하면 내부 서브그래프 상세 정보를 볼 수 있습니다

#### Python Code — Python 코드 생성
현재 노드 그래프를 실행 가능한 Python(OpenCV) 코드로 자동 변환합니다.
- 노드의 토폴로지 순서에 따라 코드가 생성됩니다
- 필요한 라이브러리를 자동으로 감지하여 import문과 pip 설치 명령어를 포함합니다
- 지원 라이브러리: OpenCV(cv2), NumPy, MediaPipe, YOLO(ultralytics), Tesseract OCR, PaddleOCR 등
- 코드 미리보기 창에서 확인 후 클립보드 복사 또는 .py 파일로 저장할 수 있습니다

#### C# Code — C# 코드 생성
현재 노드 그래프를 실행 가능한 C#(OpenCvSharp) 콘솔 애플리케이션으로 자동 변환합니다.
- OpenCvSharp 라이브러리를 사용하는 코드가 생성됩니다
- NuGet 패키지 설치 명령어(dotnet add package)가 포함됩니다
- Mat 객체의 메모리 관리(Dispose)가 자동으로 처리됩니다
- 코드 미리보기 창에서 확인 후 클립보드 복사 또는 .cs 파일로 저장할 수 있습니다

#### HelperBot — AI 챗봇 도우미
RAG(검색 증강 생성) 기반 도움말 챗봇입니다. 툴바의 HelperBot 버튼을 클릭하면 챗봇 창이 열립니다.
- MVXTester 사용법, 노드 설명, 교재 내용 등에 대해 질문할 수 있습니다
- 이미지를 첨부하여 화면 관련 질문을 할 수 있습니다 (Ctrl+V로 클립보드 붙여넣기 가능)
- 질문과 관련된 예제 프로젝트가 있으면 "예제 로드" 버튼이 표시됩니다

#### Help (F1) — 도움말
종합적인 사용자 설명서를 팝업 윈도우로 표시합니다.
- Usage 탭: 기본 사용법, 실행 모드, 키보드 단축키
- Node Reference 탭: 모든 노드의 입출력 포트와 파라미터 상세 설명
- Tutorial 탭: 단계별 실습 예제 (이미지 처리, 윤곽 검출, 카메라, 템플릿 매칭 등)

#### 테마 전환 — Dark/Light 테마
툴바 우측의 테마 아이콘(해/달)을 클릭하여 다크 테마와 라이트 테마를 전환합니다. 선택한 테마는 저장되어 다음 실행 시에도 유지됩니다.

#### 노드 팔레트 표시/숨기기
좌측 노드 팔레트 패널의 화살표 버튼(◀/▶)을 클릭하여 패널을 숨기거나 표시합니다. 패널을 숨기면 캔버스 작업 영역이 넓어집니다.

#### 프로퍼티 패널 표시/숨기기
우측 프로퍼티 패널의 화살표 버튼(▶/◀)을 클릭하여 패널을 숨기거나 표시합니다. 선택된 노드의 파라미터 편집과 결과 이미지 미리보기가 이 패널에 표시됩니다.

### 프로젝트 저장 형식

#### .mvxp — 프로젝트 아카이브 (ZIP)
그래프 JSON + 참조된 이미지/비디오 파일을 포함합니다. 공유를 위한 권장 형식입니다.

#### .mvx — 레거시 JSON 형식
그래프 정보만 포함하며, 임베디드 파일은 없습니다.
