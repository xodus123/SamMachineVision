## Chapter 8: Input/Output 노드


### 8.1 개요

  Input/Output 노드 그룹은 머신 비전 파이프라인의 시작과 끝을 담당하는 가장 기본적인 노드들로 구성됩니다. 이미지를 파일에서 읽어들이거나 카메라에서 실시간으로 캡처하여 파이프라인에 공급하는 입력 노드와, 처리 결과를 파일로 저장하거나 화면에 표시하는 출력 노드를 포함합니다. 모든 비전 처리 플로우는 반드시 하나 이상의 입력 노드에서 시작되며, 결과를 확인하기 위해 출력 노드를 연결하는 것이 일반적인 패턴입니다.


  MVXTester의 Input/Output 카테고리에는 5개의 사용자 노드가 등록되어 있습니다: Image Read(이미지 파일 읽기), Image Write(이미지 파일 저장), Video Read(비디오 프레임 읽기), Camera(통합 카메라 캡처), Image Show(이미지 화면 표시). Camera 노드는 내부적으로 USB, HIK(하이크비전), Cognex GigE 세 가지 백엔드를 통합한 통합 카메라 노드로, 연결된 모든 카메라를 하나의 디바이스 목록에서 선택할 수 있습니다. Video Read와 Camera 노드는 `IStreamingSource` 인터페이스를 구현하여 Stream 모드(F6)에서 연속 프레임을 공급할 수 있습니다.


### 8.2 Image Read 노드


#### Image Read

  파일 시스템에서 이미지를 읽어 Mat 객체로 출력하는 기본 입력 노드입니다. OpenCvSharp의 `Cv2.ImRead`를 사용하여 다양한 이미지 포맷(PNG, JPG, BMP, TIFF 등)을 지원하며, `ImreadModes.Unchanged` 모드로 원본 채널 수와 비트 깊이를 유지합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Image | Mat | 읽어들인 이미지 (원본 채널/깊이 유지) |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| File Path | FilePath | "" | - | 읽어들일 이미지 파일 경로 |


**기능 설명**

  Image Read 노드는 지정된 파일 경로에서 이미지를 읽어 Mat 객체로 변환합니다. `ImreadModes.Unchanged` 모드를 사용하므로, 그레이스케일(1채널), 컬러(3채널), 알파 채널 포함(4채널) 이미지 모두 원본 그대로 읽어들입니다. 16비트나 32비트 깊이의 이미지도 변환 없이 원본 깊이를 유지합니다.


  파일 경로가 비어 있거나 파일이 존재하지 않는 경우 에러 메시지를 설정하며, 이미지 디코딩에 실패한 경우에도 적절한 에러를 보고합니다. 정상적으로 읽어들인 이미지는 출력 포트로 전달됨과 동시에 노드 미리보기(Preview)로도 설정되어, 에디터에서 직접 이미지를 확인할 수 있습니다.


  이 노드는 파이프라인의 가장 일반적인 시작점으로, 단일 실행(F5 Runtime)과 강제 실행(Ctrl+F5) 모두에서 사용됩니다. 비디오나 카메라 캡처가 아닌 정적 이미지 분석 시나리오에 적합합니다.


**응용 분야**


    - 정적 이미지 검사: 저장된 이미지 파일을 기반으로 한 품질 검사 파이프라인 구축

    - 알고리즘 개발: 고정된 테스트 이미지로 비전 알고리즘 파라미터 튜닝

    - 배치 처리: 여러 이미지 파일을 순차적으로 로드하여 일괄 처리

    - 기준 이미지 로드: 템플릿 매칭이나 비교 검사에서 레퍼런스 이미지 제공


### 8.3 Image Write 노드


#### Image Write

  입력으로 받은 이미지를 지정된 파일 경로에 저장하는 출력 노드입니다. PNG, JPG, BMP, TIFF 네 가지 포맷을 지원하며, 선택한 포맷에 따라 자동으로 파일 확장자를 설정합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 저장할 이미지 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| File Path | FilePath | "" | - | 출력 파일 경로 |
| Format | ImageFormat | PNG | PNG, JPG, BMP, TIFF | 이미지 저장 포맷 |


**기능 설명**

  Image Write 노드는 파이프라인에서 처리된 이미지를 파일로 저장합니다. Format 속성에서 선택한 포맷(PNG, JPG, BMP, TIFF)에 따라 파일 확장자가 자동으로 결정되므로, 사용자가 지정한 경로의 확장자와 관계없이 올바른 포맷으로 저장됩니다. 예를 들어 파일 경로가 "output.bmp"이고 포맷이 PNG인 경우, 실제 저장 파일은 "output.png"이 됩니다.


  출력 디렉토리가 존재하지 않는 경우 자동으로 디렉토리를 생성합니다. 입력 이미지가 없거나 비어 있는 경우, 또는 파일 경로가 지정되지 않은 경우에는 에러를 보고합니다. OpenCvSharp의 `Cv2.ImWrite`를 사용하여 실제 저장을 수행하며, 저장 완료 후 노드 미리보기에도 이미지를 표시합니다.


  이 노드는 검사 결과 이미지 저장, 중간 처리 결과 기록, 디버깅용 이미지 출력 등 다양한 시나리오에서 활용됩니다. 특히 자동화된 검사 라인에서 불량 이미지를 자동으로 저장하는 용도로 자주 사용됩니다.


**응용 분야**


    - 검사 결과 저장: 처리 완료된 이미지를 시각화와 함께 파일로 보관

    - 불량 이미지 기록: 검사 불합격 이미지를 자동으로 저장하여 추후 분석

    - 중간 결과 디버깅: 파이프라인 중간 단계의 이미지를 파일로 저장하여 확인

    - 보고서 생성: 검사 결과 이미지를 PNG/JPG로 저장하여 품질 보고서에 첨부


### 8.4 Video Read 노드


#### Video Read

  비디오 파일에서 프레임을 순차적으로 읽어 출력하는 스트리밍 소스 노드입니다. `IStreamingSource` 인터페이스를 구현하여 Stream 모드(F6)에서 연속적으로 프레임을 공급하며, 비디오의 끝에 도달하면 자동으로 처음으로 되돌아가 반복 재생합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Frame | Mat | 현재 프레임 이미지 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| File Path | FilePath | "" | - | 비디오 파일 경로 |


**기능 설명**

  Video Read 노드는 OpenCvSharp의 `VideoCapture`를 사용하여 비디오 파일을 프레임 단위로 읽어들입니다. 매 Process() 호출 시 다음 프레임을 읽어 Frame 출력 포트로 전달합니다. 비디오 파일 경로가 변경되면 기존 캡처를 해제하고 새로운 비디오 파일을 열어, 실행 중에도 다른 비디오로 전환할 수 있습니다.


  비디오의 마지막 프레임까지 읽은 후에는 자동으로 프레임 위치를 0으로 리셋하여 처음부터 다시 재생합니다. 이러한 루프 재생 기능은 Stream 모드에서 알고리즘을 지속적으로 테스트할 때 유용합니다. 프레임 읽기에 실패한 경우 리셋 후 재시도하며, 재시도마저 실패하면 에러를 보고합니다.


  `IStreamingSource` 인터페이스를 구현하므로, Stream 모드(F6)에서 약 30fps로 자동 반복 실행됩니다. Cleanup 시에는 VideoCapture 리소스를 해제하여 메모리 누수를 방지합니다. AVI, MP4, MOV 등 OpenCV가 지원하는 모든 비디오 코덱을 사용할 수 있습니다.


**응용 분야**


    - 비디오 기반 검사: 녹화된 영상에서 프레임 단위 분석 수행

    - 알고리즘 검증: 카메라 없이 비디오 파일로 실시간 처리 알고리즘 테스트

    - 시연 및 데모: 사전 녹화 영상을 활용한 비전 시스템 데모

    - 교육용: 다양한 비디오 샘플로 비전 알고리즘 학습


### 8.5 Camera 노드


#### Camera

  USB 웹캠, HIK(하이크비전) 산업용 카메라, Cognex GigE 카메라를 하나의 통합 인터페이스로 제공하는 카메라 노드입니다. 세 가지 카메라 백엔드를 자동으로 탐색하여 연결 가능한 모든 카메라를 단일 디바이스 목록에 표시합니다.


      Trigger int


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Trigger | int | 소프트웨어 트리거 입력 (HIK/Cognex 소프트웨어 트리거 모드에서 사용) |
| Output | Frame | Mat | 캡처된 프레임 이미지 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Camera | DeviceList | -1 | - | 카메라 디바이스 선택 (자동 탐색 결과) |
| Width | int | 640 | 0 ~ 10000 | 캡처 폭 (0 = 기본값/최대) |
| Height | int | 480 | 0 ~ 10000 | 캡처 높이 (0 = 기본값/최대) |
| FPS | int | 30 | 1 ~ 120 | 목표 프레임 레이트 (USB만 해당) |
| USB Backend | UsbCameraBackend | DirectShow | DirectShow, MSMF, Auto | 캡처 백엔드 API (USB만 해당) |
| Trigger Mode | CameraTriggerMode | Continuous | Continuous, Software, Hardware | 트리거 모드 (HIK/Cognex) |
| Exposure Time | double | 10000.0 | 16.0 ~ 10000000.0 | 노출 시간 (마이크로초, HIK/Cognex) |
| Pixel Format | CameraPixelFormat | Auto | Auto, Mono8, BayerRG8, BayerGR8, BayerBG8, BayerGB8, RGB8, BGR8, Color24, YUV422_8 | 픽셀 포맷 (HIK/Cognex) |
| Gain | double | 0.0 | 0.0 ~ 20.0 | 아날로그 게인 dB (HIK) |
| Auto Exposure | bool | false | - | 자동 노출 활성화 (HIK) |
| Auto Gain | bool | false | - | 자동 게인 활성화 (HIK) |
| Gamma Enable | bool | false | - | 감마 보정 활성화 (HIK) |
| Gamma | double | 0.7 | 0.1 ~ 4.0 | 감마 값 (HIK) |
| Reverse X | bool | false | - | 수평 반전 (HIK) |
| Reverse Y | bool | false | - | 수직 반전 (HIK) |
| Brightness | double | 0.5 | 0.0 ~ 1.0 | 밝기 (Cognex) |


**기능 설명**

  Camera 노드는 MVXTester의 통합 카메라 인터페이스입니다. 내부적으로 세 가지 백엔드 노드(UsbCameraNode, HikCameraNode, CognexGigECameraNode)를 인스턴스화하여 각 백엔드의 `EnumerateDevices()`를 호출하고, 발견된 모든 디바이스를 하나의 통합 리스트로 병합합니다. USB 카메라는 [USB] 접두사, HIK 카메라는 [GigE] 또는 [USB3] 접두사, Cognex 카메라는 [Cognex] 접두사로 표시됩니다.


  사용자가 디바이스를 선택하면, 해당 카메라의 백엔드 타입에 따라 관련 속성만 표시되는 동적 속성 가시성 기능이 동작합니다. USB 카메라 선택 시 FPS, USB Backend 속성이 표시되고, HIK 카메라 선택 시 Trigger Mode, Exposure Time, Gain, Gamma 등의 산업용 카메라 속성이 표시됩니다. Cognex 카메라는 Trigger Mode, Exposure Time, Brightness 속성이 표시됩니다. Process() 실행 시 선택된 백엔드의 속성을 동기화한 후 해당 백엔드의 Process()를 호출하여 프레임을 캡처합니다.


  HIK 카메라 백엔드는 동적 어셈블리 로딩(Reflection)을 사용하여 MvCameraControl.Net.dll을 런타임에 로드합니다. 이를 통해 SDK가 설치되지 않은 환경에서도 앱이 정상적으로 실행되며, GigE 카메라의 서브넷 불일치 시 자동 ForceIP 기능도 내장되어 있습니다. Cognex 백엔드도 마찬가지로 VisionPro SDK를 동적으로 로드하며, Bayer 디모자이킹과 다양한 픽셀 포맷 변환을 자동으로 처리합니다.


**응용 분야**


    - 실시간 비전 검사: 산업용 카메라에서 실시간 이미지를 캡처하여 인라인 품질 검사

    - 멀티 카메라 시스템: 여러 Camera 노드를 사용하여 다각도 검사 구현

    - 프로토타이핑: USB 웹캠으로 빠르게 비전 알고리즘을 프로토타이핑한 후 산업용 카메라로 전환

    - 하드웨어 트리거 연동: 외부 센서 신호에 동기화된 이미지 캡처


### 8.6 Image Show 노드


#### Image Show

  입력 이미지를 OpenCV HighGUI 창에 표시하는 출력 노드입니다. Runtime 모드(F5)에서만 창이 생성되며, 마우스 및 키보드 이벤트를 `RuntimeEventBus`를 통해 Event 노드로 전달하여 상호작용 기능을 제공합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 표시할 이미지 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Window Name | string | "Image" | - | 표시 창의 제목 |


**기능 설명**

  Image Show 노드는 입력으로 받은 이미지를 OpenCV HighGUI 창에 표시합니다. OpenCV의 HighGUI는 모든 ImShow/WaitKey/SetMouseCallback 호출이 동일 스레드에서 이루어져야 하는 제약이 있기 때문에, MVXTester는 `ImageShowManager`라는 싱글턴 관리자를 사용하여 모든 Image Show 노드의 창을 하나의 전용 디스플레이 스레드에서 처리합니다. 여러 Image Show 노드가 동시에 사용되어도 스레드 안전성이 보장됩니다.


  Runtime 모드(F5 Execute)에서만 실제 OpenCV 창이 생성되며, 에디터에서는 노드의 미리보기 패널에만 이미지가 표시됩니다. OpenCV 창에서 발생하는 마우스 이벤트(클릭, 이동, 휠 등)는 `RuntimeEventBus.RaiseMouseEvent()`를 통해 전역 이벤트 버스로 발행되며, MouseEventNode나 MouseRoiNode 등의 이벤트 노드에서 이를 수신하여 인터랙티브 기능을 구현할 수 있습니다. 키보드 이벤트도 마찬가지로 `RuntimeEventBus.RaiseKeyEvent()`를 통해 발행됩니다.


  Cleanup 시에는 해당 창의 등록을 해제하며, ImageShowManager가 디스플레이 스레드에서 안전하게 창을 파괴합니다. 모든 창이 제거되면 디스플레이 스레드도 자동으로 종료됩니다.


**응용 분야**


    - 결과 시각화: 비전 처리 결과를 실시간으로 모니터링

    - 인터랙티브 ROI 설정: 마우스 이벤트와 연동하여 관심 영역을 직접 지정

    - 다중 뷰: 여러 Image Show 노드를 사용하여 원본/처리 결과를 동시에 비교

    - 디버깅: 파이프라인 중간 단계의 이미지를 별도 창에 표시하여 확인


## Chapter 9: Value 노드


### 9.1 개요

  Value 노드 그룹은 비전 파이프라인에서 사용되는 기본 데이터 값을 생성하고 조작하는 노드들로 구성됩니다. 정수, 실수, 문자열, 불리언 등의 기본 자료형 상수 노드부터, OpenCV의 Point, Size, Rect, Scalar 같은 구조체 생성 노드, 그리고 산술 연산, 비교 연산, 논리 연산 등의 데이터 처리 노드까지 포함합니다. 이 노드들은 다른 노드의 파라미터를 동적으로 제공하거나, 조건부 로직을 구현하는 데 핵심적인 역할을 합니다.


  Value 카테고리에는 총 15개의 노드가 등록되어 있습니다: 기본 자료형 상수 노드 5개(Integer, Float, Double, String, Bool), OpenCV 구조체 노드 4개(Point, Size, Scalar, Rect), 연산 노드 3개(Math Operation, Comparison, Logic Gate), 유틸리티 노드 3개(List Create, String Format, Print). 상수 노드들은 속성에서 값을 설정하여 출력 포트로 전달하는 단순한 구조이고, 연산 노드들은 입력 포트의 값을 받아 계산 결과를 출력합니다. Print 노드는 어떤 타입의 데이터든 포맷팅하여 텍스트로 표시하는 범용 디버깅 도구입니다.


### 9.2 Integer 노드


#### Integer

  정수 상수 값을 출력하는 가장 기본적인 값 노드입니다. 속성에서 설정한 정수 값을 출력 포트로 전달합니다.


      Value int


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Value | int | 정수 값 출력 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Value | int | 0 | int.MinValue ~ int.MaxValue | 출력할 정수 값 |


**기능 설명**

  Integer 노드는 속성 패널에서 설정한 정수 값을 그대로 출력 포트에 전달하는 단순한 상수 노드입니다. Process() 실행 시 항상 에러 없이 현재 속성 값을 출력합니다.


  이 노드는 다른 노드의 정수형 입력 포트에 고정 값을 공급할 때 사용됩니다. 예를 들어 임계값(threshold), 커널 크기, 반복 횟수 등의 파라미터를 동적으로 변경 가능한 형태로 제공할 수 있습니다. 속성 값을 변경하면 다음 실행에서 새로운 값이 반영되므로, 런타임 중에도 파라미터를 조정할 수 있습니다.


**응용 분야**


    - 파라미터 공급: 필터 커널 크기, 임계값 등 정수형 파라미터 제공

    - 카운터/인덱스: 리스트 인덱스, 반복 횟수 등 지정

    - 좌표 값: 특정 픽셀 위치의 X, Y 좌표 지정


### 9.3 Float 노드


#### Float

  단정밀도 부동소수점(float) 상수 값을 출력하는 노드입니다.


      Value float


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Value | float | 단정밀도 부동소수점 값 출력 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Value | float | 0 | float.MinValue ~ float.MaxValue | 출력할 float 값 |


**기능 설명**

  Float 노드는 Integer 노드와 동일한 구조를 가지며, 단정밀도 부동소수점 값을 출력합니다. 소수점 이하 값이 필요한 파라미터에 사용됩니다.


  float 타입은 약 7자리의 유효 숫자를 제공하며, 더 높은 정밀도가 필요한 경우 Double 노드를 사용하는 것이 적합합니다. 주로 OpenCV 함수에서 float 타입 파라미터를 요구하는 경우에 사용됩니다.


**응용 분야**


    - 스케일 팩터: 이미지 리사이즈 비율, 알파 블렌딩 가중치 등

    - OpenCV 파라미터: float 타입을 요구하는 필터/변환 파라미터 공급

    - 좌표 보간: 서브픽셀 정밀도의 좌표 값 제공

  배정밀도 부동소수점(double) 상수 값을 출력하는 노드입니다.


      Value double


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Value | double | 배정밀도 부동소수점 값 출력 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Value | double | 0.0 | double.MinValue ~ double.MaxValue | 출력할 double 값 |


**기능 설명**

  Double 노드는 배정밀도 부동소수점 값을 출력합니다. 약 15~17자리의 유효 숫자를 제공하므로 높은 정밀도가 필요한 계산에 적합합니다.


  Math Operation 노드의 입력 타입이 double이므로, 산술 연산 파이프라인에서 가장 자주 사용되는 상수 노드입니다. Comparison 노드와 조합하여 임계값 기반 판정 로직을 구현할 때도 활용됩니다.


**응용 분야**


    - 정밀 파라미터: 높은 정밀도가 요구되는 측정/계산 파라미터

    - 산술 연산: Math Operation 노드와 조합한 수치 계산

    - 임계값: Comparison 노드의 비교 기준값 공급

  String Format 노드의 포맷 템플릿과 조합하거나, Print 노드의 라벨로 활용할 수 있습니다. 또한 파일 경로를 동적으로 구성하는 데에도 유용합니다.


**응용 분야**

    - 패턴 매칭: 정규 표현식이나 검색 패턴 문자열 제공

**응용 분야**


    - 좌표 지정: 특정 픽셀 위치 또는 관심 지점 지정

    - 도형 그리기: Draw 노드에서 기준점/시작점 공급

    - 앵커 설정: 모폴로지 연산 등의 앵커 포인트 지정

**포트 상세**

**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |

  예를 들어 Gaussian Blur의 커널 크기나, Resize 노드의 목표 크기를 지정할 때 이 노드를 연결하여 크기 값을 공급할 수 있습니다.

    - 이미지 리사이즈: 목표 해상도 크기 지정

    - 윈도우 크기: 슬라이딩 윈도우 등의 크기 파라미터


### 9.9 Scalar 노드


#### Scalar

  OpenCV의 4채널 스칼라 값(`Scalar`)을 생성하여 출력하는 노드입니다. 색상 지정(BGR+Alpha) 등에 주로 사용됩니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Output | Scalar | Scalar | 4채널 스칼라 값 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| V0 | double | 0.0 | 0.0 ~ 255.0 | 컴포넌트 0 (Blue) |
| V1 | double | 0.0 | 0.0 ~ 255.0 | 컴포넌트 1 (Green) |
| V2 | double | 0.0 | 0.0 ~ 255.0 | 컴포넌트 2 (Red) |
| V3 | double | 0.0 | 0.0 ~ 255.0 | 컴포넌트 3 (Alpha) |


**기능 설명**

  Scalar 노드는 V0~V3 네 개의 double 속성으로 OpenCvSharp의 `Scalar` 구조체를 생성합니다. OpenCV에서 Scalar는 주로 색상 값을 표현하는 데 사용되며, BGR 색상 공간에서 V0=Blue, V1=Green, V2=Red, V3=Alpha로 매핑됩니다.


  InRange 노드의 하한/상한 경계값, Draw 노드의 색상, 이미지 마스킹의 채우기 색상 등에 활용됩니다. 각 컴포넌트의 범위는 0.0~255.0으로 설정되어 있어 8비트 이미지의 색상 값을 직관적으로 지정할 수 있습니다.


**응용 분야**


    - 색상 지정: Draw 노드의 도형 색상, 텍스트 색상 지정

    - 범위 경계: InRange 색상 필터링의 상한/하한 값

    - 채우기 값: 이미지의 특정 영역을 지정 색상으로 채우기

    - 배경색: 빈 이미지 생성 시 초기 색상 설정

### 9.11 Math Operation 노드


#### Math Operation

  두 개의 double 값에 대해 산술 연산을 수행하는 연산 노드입니다. 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지, 거듭제곱, 최솟값, 최댓값 총 8가지 연산을 지원합니다.


      A double

      B double


      Result double


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | A | double | 첫 번째 피연산자 |
| Input | B | double | 두 번째 피연산자 |
| Output | Result | double | 연산 결과 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Operation | MathOp | Add | Add, Subtract, Multiply, Divide, Modulo, Power, Min, Max | 수행할 산술 연산 종류 |


**기능 설명**

  Math Operation 노드는 입력 포트 A와 B의 값에 대해 선택한 연산을 수행하여 결과를 출력합니다. 지원되는 연산은 Add(A+B), Subtract(A-B), Multiply(A*B), Divide(A/B), Modulo(A%B), Power(A^B), Min(A,B 중 작은 값), Max(A,B 중 큰 값) 8가지입니다.


  Divide와 Modulo 연산에서 B가 0인 경우 결과를 0으로 반환하여 0으로 나누기 에러를 방지합니다. Power 연산은 `Math.Pow(A, B)`를 사용합니다. 여러 Math Operation 노드를 체인으로 연결하면 복잡한 수식을 구현할 수 있습니다.


  측정 결과의 단위 변환(픽셀 → mm), 검사 판정 기준값 계산, 통계 값 처리 등에 활용됩니다.


**응용 분야**


    - 단위 변환: 픽셀 단위의 측정값을 실제 단위(mm 등)로 변환

    - 보정 계산: 오프셋, 스케일 보정 등 선형 변환

    - 통계 처리: 최솟값/최댓값 비교, 범위 계산

    - 수식 구현: 여러 노드를 조합하여 복잡한 수학 공식 표현


### 9.12 Comparison 노드


#### Comparison

  두 개의 double 값을 비교하여 불리언 결과를 출력하는 노드입니다. 6가지 비교 연산자를 지원합니다.


      A double

      B double


      Result bool


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | A | double | 비교할 첫 번째 값 |
| Input | B | double | 비교할 두 번째 값 |
| Output | Result | bool | 비교 결과 (true/false) |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Operator | ComparisonOp | GreaterThan | GreaterThan, LessThan, GreaterOrEqual, LessOrEqual, Equal, NotEqual | 비교 연산자 |


**기능 설명**

  Comparison 노드는 입력 A와 B를 선택한 연산자로 비교하여 불리언 결과를 출력합니다. GreaterThan(A > B), LessThan(A < B), GreaterOrEqual(A ≥ B), LessOrEqual(A ≤ B), Equal(A = B), NotEqual(A &ne; B) 6가지 비교를 지원합니다.


  Equal과 NotEqual 비교는 부동소수점 정밀도를 고려하여 `Math.Abs(A - B)`와 `double.Epsilon`을 비교하는 방식으로 구현되어 있습니다. 이를 통해 부동소수점 연산의 미세한 오차로 인한 잘못된 비교 결과를 방지합니다.


  검사 판정 로직에서 핵심적인 역할을 합니다. 예를 들어 측정값이 허용 공차 범위 내인지 확인하거나, 검출 결과의 신뢰도가 임계값 이상인지 판단하는 데 사용됩니다.


**응용 분야**


    - 품질 판정: 측정값이 허용 범위(공차) 내인지 판단

    - 임계값 비교: 검출 신뢰도, 면적, 길이 등이 기준을 충족하는지 확인

    - 조건부 분기: 비교 결과를 Logic Gate와 조합하여 복합 조건 구현

    - 경고 트리거: 특정 값이 한계를 초과할 때 경고 발생


### 9.13 Logic Gate 노드


#### Logic Gate

  두 개의 불리언 입력에 대해 논리 게이트 연산을 수행하는 노드입니다. AND, OR, XOR, NAND, NOR 5가지 게이트를 지원합니다.


      A bool

      B bool


      Result bool


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | A | bool | 첫 번째 불리언 입력 |
| Input | B | bool | 두 번째 불리언 입력 |
| Output | Result | bool | 논리 연산 결과 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Gate | LogicGate | AND | AND, OR, XOR, NAND, NOR | 논리 게이트 종류 |


**기능 설명**

  Logic Gate 노드는 두 불리언 입력에 대해 선택한 논리 게이트 연산을 수행합니다. AND(A && B), OR(A || B), XOR(A ^ B), NAND(!(A && B)), NOR(!(A || B)) 5가지 기본 논리 게이트를 지원합니다.


  Comparison 노드의 출력을 여러 Logic Gate 노드로 조합하면 복합 판정 조건을 구현할 수 있습니다. 예를 들어 "면적이 100 이상 AND 길이가 50 이하"와 같은 다중 조건 검사를 노드 그래프로 시각적으로 표현할 수 있습니다.


  여러 Logic Gate 노드를 캐스케이드로 연결하면 3개 이상의 조건을 조합하는 것도 가능합니다. NAND와 NOR 게이트는 NOT 연산이 필요한 경우에도 활용할 수 있습니다.


**응용 분야**


    - 복합 판정: 여러 비교 조건을 AND/OR로 조합하여 최종 합격/불합격 판정

    - 인터락 로직: 여러 조건이 모두 만족될 때만 동작하는 안전 인터락 구현

    - 예외 처리: XOR, NAND 등으로 특수 조건 감지

    - NOT 대체: NAND/NOR 게이트를 활용한 논리 반전


### 9.14 List Create 노드


#### List Create

  여러 개의 입력 값을 하나의 리스트로 수집하는 노드입니다. 최대 8개의 입력을 받아 `List<object>`로 출력합니다.


      Item 0 object

      Item 1 object

      ... object

      Item 7 object


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Item 0 | object | 리스트에 추가할 첫 번째 항목 |
| Input | Item 1 | object | 리스트에 추가할 두 번째 항목 |
| Input | Item 2 | object | 리스트에 추가할 세 번째 항목 |
| Input | Item 3 | object | 리스트에 추가할 네 번째 항목 |
| Input | Item 4 | object | 리스트에 추가할 다섯 번째 항목 |
| Input | Item 5 | object | 리스트에 추가할 여섯 번째 항목 |
| Input | Item 6 | object | 리스트에 추가할 일곱 번째 항목 |
| Input | Item 7 | object | 리스트에 추가할 여덟 번째 항목 |
| Output | List | List<object> | 수집된 항목들의 리스트 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| 속성 없음 |


**기능 설명**

  List Create 노드는 최대 8개의 입력 포트(Item 0 ~ Item 7)에 연결된 값을 순서대로 수집하여 하나의 `List<object>`로 출력합니다. 모든 입력 포트의 타입이 `object`이므로 정수, 실수, 문자열, Mat 등 어떤 타입의 데이터든 혼합하여 리스트에 포함할 수 있습니다.


  연결되지 않은 포트나 null 값을 가진 포트는 자동으로 건너뛰어, 실제 연결된 항목만으로 리스트가 구성됩니다. 예를 들어 Item 0과 Item 2만 연결하면 2개의 항목으로 구성된 리스트가 생성됩니다.


**응용 분야**


    - 데이터 수집: 여러 노드의 출력을 하나의 리스트로 통합

    - 배치 처리: 여러 이미지나 파라미터를 리스트로 묶어 일괄 처리

    - 결과 집계: 여러 검사 결과를 하나의 리스트로 수집하여 후속 분석


### 9.15 String Format 노드


#### String Format

  포맷 문자열 템플릿에 인자를 삽입하여 결과 문자열을 생성하는 노드입니다. C#의 `string.Format`과 동일한 {0}, {1} 등의 플레이스홀더를 사용합니다.


      Arg 0 object

      Arg 1 object

      Arg 2 object

      Arg 3 object


      Result string


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Arg 0 | object | 포맷 인자 {0}에 삽입할 값 |
| Input | Arg 1 | object | 포맷 인자 {1}에 삽입할 값 |
| Input | Arg 2 | object | 포맷 인자 {2}에 삽입할 값 |
| Input | Arg 3 | object | 포맷 인자 {3}에 삽입할 값 |
| Output | Result | string | 포맷팅된 결과 문자열 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Format | string | "{0}" | - | 포맷 템플릿 ({0}, {1} 등 플레이스홀더 사용) |


**기능 설명**

  String Format 노드는 C#의 `string.Format()` 메서드를 노드 인터페이스로 제공합니다. Format 속성에 "{0}, {1}" 형태의 템플릿 문자열을 작성하고, 각 플레이스홀더에 대응하는 Arg 입력 포트에 값을 연결하면 포맷팅된 문자열이 Result 포트로 출력됩니다.


  연결되지 않은 인자 포트는 빈 문자열("")로 대체되므로, 일부 인자만 사용하는 포맷도 문제없이 동작합니다. 최대 4개의 인자({0}~{3})를 지원하며, C#의 포맷 지정자(예: {0:F2}, {1:D4})도 사용할 수 있어 숫자의 소수점 자릿수나 패딩을 제어할 수 있습니다.


**응용 분야**


    - 결과 텍스트 생성: "합격: {0}개, 불합격: {1}개" 형태의 검사 결과 메시지 구성

    - 파일명 동적 생성: "result_{0}_{1}.png" 형태의 동적 파일 경로 구성

    - 로그 메시지: 측정값과 설명을 조합한 로그 문자열 생성

    - 라벨 텍스트: 이미지 오버레이용 동적 텍스트 구성


### 9.16 Print 노드


#### Print

  어떤 타입의 데이터든 포맷팅하여 텍스트로 표시하는 범용 디버깅 노드입니다. 최대 4개의 입력을 동시에 받아 각각의 값을 타입에 따라 적절한 형식으로 표시합니다. 이미지 기반 미리보기 대신 WPF TextBlock 기반의 텍스트 미리보기를 사용합니다.


      In 0 object

      In 1 object

      In 2 object

      In 3 object


      Text string


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | In 0 | object | 표시할 첫 번째 값 (모든 타입 가능) |
| Input | In 1 | object | 표시할 두 번째 값 |
| Input | In 2 | object | 표시할 세 번째 값 |
| Input | In 3 | object | 표시할 네 번째 값 |
| Output | Text | string | 포맷팅된 전체 텍스트 |


**속성**


| 속성 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Max Items | int | 20 | 1 ~ 200 | 배열/리스트 표시 시 최대 항목 수 |
| Show Type | bool | true | - | 데이터 타입 이름 표시 여부 |
| Label 0 | string | "" | - | Input 0의 커스텀 라벨 (빈 경우 "In 0" 사용) |
| Label 1 | string | "" | - | Input 1의 커스텀 라벨 |
| Label 2 | string | "" | - | Input 2의 커스텀 라벨 |
| Label 3 | string | "" | - | Input 3의 커스텀 라벨 |


**기능 설명**

  Print 노드는 MVXTester의 범용 디버깅 도구로, 모든 데이터 타입을 사람이 읽기 쉬운 텍스트 형식으로 변환하여 표시합니다. 노드의 미리보기 영역에 `SetTextPreview()`를 통해 텍스트를 직접 표시하므로, 이미지가 아닌 데이터도 노드에서 바로 확인할 수 있습니다.


  타입별 전용 포맷터를 내장하고 있어, 각 데이터 타입에 최적화된 표시를 제공합니다. Mat 이미지는 크기, 채널 수, 타입, 중심 픽셀 값을 표시합니다. Point, Rect, Size, Scalar 등의 OpenCV 구조체는 각 필드를 명확하게 표시합니다. 배열(int[], double[], Point[], Rect[] 등)은 Count와 함께 각 요소를 인덱스 번호와 함께 나열하며, Max Items 속성으로 표시 항목 수를 제한합니다. byte[]는 16진수 형식으로 표시됩니다.


  연결된 포트만 표시에 포함되며, 각 입력에 커스텀 라벨을 지정할 수 있어 여러 값을 동시에 모니터링할 때 식별이 용이합니다. Show Type 옵션을 켜면 각 값의 타입 이름(int, Mat, Point 등)도 함께 표시됩니다. 포맷팅된 텍스트는 Text 출력 포트로도 전달되어 후속 노드에서 활용할 수 있습니다.


**응용 분야**


    - 값 모니터링: 파이프라인 중간 결과값을 실시간으로 확인

    - 디버깅: 연산 결과, 검출 좌표, 매칭 점수 등을 가시적으로 확인

    - 이미지 정보 확인: Mat 이미지의 크기, 채널, 타입 정보 파악

    - 배열 내용 확인: 검출 결과 배열의 내용을 항목별로 확인
