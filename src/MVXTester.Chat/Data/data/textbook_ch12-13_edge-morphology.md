## Chapter 12: Edge Detection 노드


### 12.1 개요

  Edge Detection 노드 그룹은 이미지에서 객체의 윤곽선과 경계를 검출하는 데 사용되는 에지 검출 알고리즘 노드들로 구성됩니다. 에지 검출은 머신 비전의 핵심 전처리 단계로, 이미지 내에서 밝기 값이 급격하게 변하는 영역을 식별하여 객체의 형상, 경계, 구조적 특징을 추출합니다. 이러한 에지 정보는 이후 단계의 윤곽선 분석, 형상 매칭, 객체 인식 등에 필수적인 입력 데이터로 활용됩니다.


  MVXTester의 Edge Detection 카테고리에는 4개의 노드가 등록되어 있습니다: Canny Edge(캐니 에지 검출), Sobel Edge(소벨 미분 에지), Laplacian Edge(라플라시안 에지), Scharr Edge(샤르 미분 에지). 각 노드는 서로 다른 수학적 접근 방식을 사용하여 에지를 검출하며, 검사 대상과 요구 정밀도에 따라 적절한 노드를 선택할 수 있습니다. Canny는 다단계 필터링을 통한 정밀한 에지 검출, Sobel과 Scharr는 방향별 미분 기반 에지 검출, Laplacian은 2차 미분 기반 에지 검출을 제공합니다.


### 12.2 Canny Edge 노드


#### Canny Edge

  Canny 에지 검출 알고리즘을 적용하여 이미지에서 에지를 검출하는 노드입니다. 이중 임계값(hysteresis thresholding)을 사용하여 강한 에지와 약한 에지를 구분하고, 비최대 억제(non-maximum suppression)를 통해 얇고 정밀한 에지를 생성합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 에지를 검출할 입력 이미지 |
| Output | Result | Mat | 에지 검출 결과 이진 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Threshold 1 | double | 100.0 | 0.0 ~ 1000.0 | 히스테리시스 이중 임계값 중 첫 번째 (하한) 임계값 |
| Threshold 2 | double | 200.0 | 0.0 ~ 1000.0 | 히스테리시스 이중 임계값 중 두 번째 (상한) 임계값 |
| Aperture Size | int | 3 | 3 ~ 7 | 내부 Sobel 연산자의 커널 크기 (3, 5, 7 중 홀수) |


**기능 설명**

  Canny Edge 노드는 John Canny가 제안한 다단계 에지 검출 알고리즘을 구현합니다. 이 알고리즘은 세 단계로 구성됩니다: 먼저 Sobel 연산자를 사용하여 그래디언트 크기와 방향을 계산하고, 비최대 억제(non-maximum suppression)를 적용하여 에지 후보를 얇게 만든 후, 이중 임계값 기법을 통해 최종 에지를 결정합니다. Aperture Size 속성은 내부 Sobel 연산에 사용되는 커널 크기를 지정하며, 값은 반드시 홀수(3, 5, 7)여야 합니다. 짝수가 입력되면 자동으로 홀수로 보정됩니다.


  Threshold 1과 Threshold 2는 에지 강도에 대한 이중 임계값을 정의합니다. 그래디언트 크기가 Threshold 2보다 큰 픽셀은 강한 에지로 즉시 채택되고, Threshold 1보다 작은 픽셀은 즉시 제거됩니다. 두 임계값 사이의 픽셀은 강한 에지와 연결되어 있는 경우에만 에지로 채택됩니다. 일반적으로 Threshold 2는 Threshold 1의 2~3배로 설정하는 것이 권장됩니다.


  출력 결과는 에지 위치만 나타내는 이진 이미지(0 또는 255)이며, 에지 픽셀은 흰색(255), 배경은 검은색(0)으로 표현됩니다. Canny 알고리즘은 노이즈에 강하고 얇은 에지를 생성하므로, 정밀한 윤곽선 추출이 필요한 산업 검사에서 가장 널리 사용되는 에지 검출 방법입니다.


**응용 분야**


    - 부품 윤곽 검출: 제조 부품의 외곽선을 정밀하게 추출하여 치수 측정에 활용

    - 결함 검사: 표면 크랙, 스크래치 등 선형 결함의 에지를 검출하여 불량 판정

    - PCB 패턴 검사: 인쇄 회로 기판의 배선 패턴 경계를 검출하여 단선/단락 확인

    - 가이드 정렬: 에지 정보를 기반으로 부품의 위치와 각도를 정밀 측정

    - 텍스트 영역 검출: 문서 이미지에서 텍스트 영역의 경계를 식별하여 OCR 전처리에 활용


### 12.3 Sobel Edge 노드


#### Sobel Edge

  Sobel 연산자를 사용하여 이미지의 1차 미분(그래디언트)을 계산하는 에지 검출 노드입니다. X축과 Y축 방향의 미분 차수를 개별적으로 설정할 수 있어, 수평 방향 또는 수직 방향 에지를 선택적으로 검출할 수 있습니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 에지를 검출할 입력 이미지 |
| Output | Result | Mat | Sobel 에지 검출 결과 이미지 (절대값 변환 적용) |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| X Order | int | 1 | 0 ~ 3 | X 방향 미분 차수 |
| Y Order | int | 0 | 0 ~ 3 | Y 방향 미분 차수 |
| Kernel Size | int | 3 | 1 ~ 7 | Sobel 커널 크기 (홀수) |


**기능 설명**

  Sobel Edge 노드는 Sobel 연산자를 사용하여 이미지의 공간적 그래디언트를 계산합니다. X Order(dx)와 Y Order(dy)는 각각 수평과 수직 방향의 미분 차수를 지정합니다. 예를 들어 dx=1, dy=0이면 수직 방향 에지(수평 밝기 변화)를 검출하고, dx=0, dy=1이면 수평 방향 에지(수직 밝기 변화)를 검출합니다. 두 값을 모두 1로 설정하면 대각선 방향의 에지도 감지할 수 있습니다.


  내부적으로 `Cv2.Sobel`을 호출하여 결과를 16비트 부호 있는 정수(CV_16S)로 먼저 계산한 후, `Cv2.ConvertScaleAbs`를 적용하여 절대값을 취하고 8비트 이미지로 변환합니다. 이 과정은 음수 방향의 그래디언트도 양수로 변환하여 에지로 표현합니다. Kernel Size는 반드시 홀수여야 하며, 짝수가 입력되면 자동으로 +1 보정됩니다.


  Sobel 연산자는 Canny와 달리 이진 에지가 아닌 그래디언트 크기를 출력하므로, 에지의 강도 정보가 보존됩니다. 이 특성을 활용하면 에지 강도에 따른 단계별 필터링이나, 특정 방향의 에지만 선택적으로 추출하는 것이 가능합니다. 머신 비전에서 부품의 방향성 패턴 검사나 텍스처 분석에 자주 활용됩니다.


**응용 분야**


    - 방향성 에지 검출: 수평 또는 수직 방향의 에지만 선택적으로 추출하여 패턴 분석

    - 그래디언트 기반 검사: 에지 강도 정보를 활용한 표면 결함의 심각도 평가

    - 텍스처 방향 분석: X/Y 미분 결과를 조합하여 재질 표면의 방향성 분석

    - 바코드/QR코드 검출: 수직 에지를 강조하여 바코드 영역을 빠르게 식별


### 12.4 Laplacian Edge 노드


#### Laplacian Edge

  Laplacian 연산자를 사용하여 이미지의 2차 미분을 계산하는 에지 검출 노드입니다. 모든 방향의 에지를 동시에 검출하며, 밝기 변화의 급변점(zero-crossing)을 기반으로 에지를 식별합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 에지를 검출할 입력 이미지 |
| Output | Result | Mat | Laplacian 에지 검출 결과 이미지 (절대값 변환 적용) |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Kernel Size | int | 3 | 1 ~ 31 | Laplacian 커널 크기 (홀수) |


**기능 설명**

  Laplacian Edge 노드는 Laplacian 연산자를 사용하여 이미지의 2차 공간 미분을 계산합니다. Sobel이 1차 미분(그래디언트)을 계산하는 것과 달리, Laplacian은 2차 미분을 계산하여 밝기 변화의 변곡점을 검출합니다. 이로 인해 방향에 무관하게 모든 에지를 동시에 검출할 수 있으나, 노이즈에 더 민감한 특성이 있습니다.


  내부적으로 `Cv2.Laplacian`을 호출하여 16비트 부호 있는 정수(CV_16S) 결과를 먼저 계산한 후, `Cv2.ConvertScaleAbs`로 절대값을 취하고 8비트 이미지로 변환합니다. Kernel Size는 홀수여야 하며, 짝수 입력 시 자동으로 +1 보정됩니다. 커널 크기가 클수록 더 넓은 영역의 밝기 변화를 감지하지만, 에지의 위치 정밀도는 감소합니다.


  Laplacian 에지 검출은 방향에 독립적이므로, 복잡한 형상의 객체나 곡선 형태의 에지를 한 번에 검출하는 데 유리합니다. 그러나 노이즈에 민감하므로, 일반적으로 Gaussian Blur 등의 평활화 필터를 전처리로 적용한 후 사용하는 것이 권장됩니다. 이를 LoG(Laplacian of Gaussian) 기법이라 합니다.


**응용 분야**


    - 전방향 에지 검출: 방향에 무관한 에지 검출이 필요한 복잡 형상 객체 분석

    - 초점 품질 측정: Laplacian 응답의 분산을 계산하여 이미지의 선명도(초점) 평가

    - 블롭 검출 전처리: LoG 기법을 활용한 특징점 및 블롭 검출의 기초 단계

    - 곡면 결함 검사: 곡선 형태의 스크래치나 크랙을 방향 무관하게 검출


### 12.5 Scharr Edge 노드


#### Scharr Edge

  Scharr 연산자를 사용하여 이미지의 1차 미분을 계산하는 에지 검출 노드입니다. Sobel 연산자의 개선된 버전으로, 작은 커널(3x3) 크기에서 더 정확한 회전 대칭 그래디언트를 제공합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 에지를 검출할 입력 이미지 |
| Output | Result | Mat | Scharr 에지 검출 결과 이미지 (절대값 변환 적용) |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| X Order | int | 1 | 0 ~ 1 | X 방향 미분 차수 (0 또는 1) |
| Y Order | int | 0 | 0 ~ 1 | Y 방향 미분 차수 (0 또는 1) |


**기능 설명**

  Scharr Edge 노드는 Scharr 필터를 사용하여 이미지의 1차 미분을 계산합니다. Scharr 연산자는 Sobel 연산자와 동일한 개념이지만, 3x3 커널에서 더 정확한 그래디언트 근사를 제공하도록 최적화된 계수를 사용합니다. Sobel의 3x3 커널이 [-1, 0, 1; -2, 0, 2; -1, 0, 1]인 반면, Scharr 커널은 [-3, 0, 3; -10, 0, 10; -3, 0, 3]으로 중심 가중치가 더 크기 때문에 회전 대칭성이 향상됩니다.


  X Order(dx)와 Y Order(dy)는 각각 0 또는 1만 설정할 수 있으며, 최소한 하나는 반드시 1이어야 합니다. 두 값이 모두 0이면 미분이 수행되지 않으므로, 노드는 에러 메시지를 표시합니다. 내부적으로 `Cv2.Scharr`를 호출하여 16비트 결과를 계산한 후, `Cv2.ConvertScaleAbs`로 절대값 변환 및 8비트 변환을 수행합니다.


  Scharr 연산자는 Sobel에 비해 작은 커널에서의 방향 정밀도가 높으므로, 고정밀 에지 방향 분석이 필요한 경우에 적합합니다. 특히 에지의 각도를 정확하게 측정해야 하는 산업 검사 응용에서, Sobel 대신 Scharr를 사용하면 더 안정적인 결과를 얻을 수 있습니다. 다만 커널 크기를 변경할 수 없으므로(항상 3x3), 넓은 범위의 에지 검출이 필요한 경우에는 Sobel 노드를 사용하는 것이 적합합니다.


**응용 분야**


    - 고정밀 에지 방향 분석: 에지 각도를 정확하게 측정해야 하는 부품 정렬 및 위치 보정

    - 미세 에지 검출: 3x3 커널에서 최적화된 계수로 미세한 밝기 변화를 정밀하게 검출

    - Sobel 대체: 3x3 커널 사용 시 Sobel보다 더 정확한 결과가 필요한 검사 시나리오

    - 그래디언트 방향 계산: X/Y 미분 결과를 조합하여 에지의 정확한 방향 벡터 산출


## Chapter 13: Morphology 및 Threshold 노드


### 13.1 개요

  Morphology 및 Threshold 노드 그룹은 이미지의 이진화(thresholding)와 형태학적 변환(morphological transformation)을 수행하는 핵심 전처리/후처리 노드들로 구성됩니다. Threshold 노드들은 그레이스케일 이미지를 이진 이미지로 변환하여 관심 영역과 배경을 분리하는 역할을 하며, Morphology 노드들은 이진화된 이미지의 형태를 다듬고 노이즈를 제거하는 역할을 합니다. 이 두 가지 처리는 머신 비전 파이프라인에서 항상 함께 사용되는 상호 보완적인 연산입니다.


  MVXTester의 Morphology 카테고리에는 3개의 노드가 등록되어 있습니다: Erode(침식), Dilate(팽창), Morphology Ex(고급 형태학 연산). Threshold 카테고리에도 3개의 노드가 등록되어 있습니다: Threshold(고정 임계값), Adaptive Threshold(적응형 임계값), Otsu Threshold(오츠 자동 임계값). 일반적인 비전 파이프라인에서는 먼저 Threshold 노드로 이미지를 이진화한 후, Morphology 노드로 형태를 정제하는 순서로 사용됩니다.


### 13.2 Morphology 노드

  형태학적 변환(Morphological Transformation)은 이미지의 형상을 기반으로 한 영상 처리 기법으로, 주로 이진 이미지에 적용됩니다. 구조 요소(Structuring Element)라 불리는 커널을 이미지 위에서 이동시키면서, 커널 영역 내 픽셀값의 최솟값(침식) 또는 최댓값(팽창)을 취하여 이미지의 형태를 변환합니다.


#### 13.2.1 Erode 노드

  침식(Erode) 연산을 수행하는 형태학적 변환 노드입니다. 구조 요소 영역 내의 최솟값을 취하여 밝은 영역을 축소하고 어두운 영역을 확장합니다. 이진 이미지에서 흰색 객체의 경계를 깎아내는 효과가 있습니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 침식 연산을 적용할 입력 이미지 |
| Output | Result | Mat | 침식 처리된 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Kernel Size | int | 3 | 1 ~ 99 | 구조 요소(Structuring Element)의 크기 |
| Iterations | int | 1 | 1 ~ 100 | 침식 연산 반복 횟수 |


**기능 설명**

  Erode 노드는 지정된 크기의 사각형(Rect) 구조 요소를 사용하여 침식 연산을 수행합니다. 내부적으로 `Cv2.GetStructuringElement(MorphShapes.Rect, ...)`를 호출하여 구조 요소를 생성하고, `Cv2.Erode`로 침식을 적용합니다. Kernel Size는 구조 요소의 너비와 높이를 동시에 지정하며, 값이 클수록 더 강한 침식 효과가 나타납니다.


  Iterations 속성은 침식 연산의 반복 횟수를 지정합니다. 반복 횟수를 높이면 커널 크기를 키우는 것과 유사한 효과를 얻을 수 있지만, 결과는 정확히 동일하지 않습니다. 큰 커널 한 번 적용보다 작은 커널 여러 번 적용이 계산 효율이 높은 경우가 있으므로, 상황에 맞게 두 속성을 조합하여 사용합니다.


  침식 연산은 이진 이미지에서 작은 흰색 노이즈 점을 제거하거나, 연결된 객체들을 분리하는 데 효과적입니다. 또한 객체의 경계를 축소하므로, Dilate(팽창) 연산과 함께 사용하면 Opening(열기) 또는 Closing(닫기) 등의 고급 형태학 연산을 구성할 수 있습니다.


**응용 분야**


    - 노이즈 제거: 이진화 후 발생하는 미세한 흰색 노이즈 점 제거

    - 객체 분리: 서로 닿아 있는 인접 객체들의 경계를 축소하여 개별 분리

    - 얇은 구조 제거: 가느다란 선이나 돌출부를 침식하여 주요 형태만 추출

    - 경계 측정: 침식 전후 이미지 차이로 객체의 경계 영역을 추출


#### 13.2.2 Dilate 노드

  팽창(Dilate) 연산을 수행하는 형태학적 변환 노드입니다. 구조 요소 영역 내의 최댓값을 취하여 밝은 영역을 확장하고 어두운 영역을 축소합니다. 이진 이미지에서 흰색 객체의 경계를 확장하는 효과가 있습니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 팽창 연산을 적용할 입력 이미지 |
| Output | Result | Mat | 팽창 처리된 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Kernel Size | int | 3 | 1 ~ 99 | 구조 요소(Structuring Element)의 크기 |
| Iterations | int | 1 | 1 ~ 100 | 팽창 연산 반복 횟수 |


**기능 설명**

  Dilate 노드는 지정된 크기의 사각형(Rect) 구조 요소를 사용하여 팽창 연산을 수행합니다. 내부적으로 `Cv2.GetStructuringElement(MorphShapes.Rect, ...)`를 호출하여 구조 요소를 생성하고, `Cv2.Dilate`로 팽창을 적용합니다. 팽창 연산은 침식의 반대 연산으로, 커널 영역 내의 최댓값을 중심 픽셀에 할당합니다.


  Kernel Size가 클수록 객체가 더 크게 확장되며, Iterations를 통해 연산을 반복 적용할 수 있습니다. 팽창은 이진 이미지에서 객체 내부의 작은 구멍(holes)을 메우거나, 가까이 있는 객체들을 연결하여 하나의 영역으로 합치는 데 효과적입니다.


  팽창 연산은 침식 연산과 쌍을 이루어 다양한 형태학적 처리를 구성합니다. 예를 들어 팽창 후 침식을 적용하면 Closing(닫기) 연산이 되어 객체 내부의 구멍을 메우면서 원래 크기를 유지하고, 침식 후 팽창을 적용하면 Opening(열기) 연산이 되어 노이즈를 제거하면서 원래 크기를 유지합니다. 이러한 복합 연산은 Morphology Ex 노드에서 직접 제공됩니다.


**응용 분야**


    - 객체 연결: 가까이 위치한 분리된 객체들을 하나의 영역으로 병합

    - 구멍 메우기: 객체 내부의 작은 검은색 구멍(holes)을 채워 연속된 영역으로 만듦

    - 끊어진 에지 연결: 불완전한 윤곽선의 끊어진 부분을 연결하여 닫힌 형태로 완성

    - 영역 확장: 관심 영역(ROI)을 일정 크기만큼 확장하여 마진을 확보


#### 13.2.3 Morphology Ex 노드

  Open, Close, Gradient, TopHat, BlackHat 등 고급 형태학적 연산을 수행하는 노드입니다. 구조 요소의 형태(사각형, 십자형, 타원형)를 선택할 수 있으며, 단일 노드로 다양한 형태학적 변환을 적용할 수 있습니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 형태학적 연산을 적용할 입력 이미지 |
| Output | Result | Mat | 형태학적 연산 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Operation | MorphOperation | Open | Open, Close, Gradient, TopHat, BlackHat | 적용할 형태학적 연산 종류 |
| Kernel Shape | MorphShape | Rect | Rect, Cross, Ellipse | 구조 요소의 형태 |
| Kernel Size | int | 5 | 1 ~ 99 | 구조 요소의 크기 |
| Iterations | int | 1 | 1 ~ 100 | 연산 반복 횟수 |


**기능 설명**

  Morphology Ex 노드는 OpenCV의 `Cv2.MorphologyEx` 함수를 래핑하여 5가지 고급 형태학적 연산을 제공합니다. **Open(열기)**은 침식 후 팽창을 수행하여 작은 노이즈를 제거하면서 객체의 원래 크기를 유지합니다. **Close(닫기)**는 팽창 후 침식을 수행하여 객체 내부의 작은 구멍을 메우면서 원래 크기를 유지합니다. **Gradient(그래디언트)**는 팽창과 침식의 차이를 계산하여 객체의 외곽선을 추출합니다.


  **TopHat(탑햇)**은 원본 이미지에서 Open 결과를 빼서, 배경보다 밝은 미세한 구조를 강조합니다. **BlackHat(블랙햇)**은 Close 결과에서 원본 이미지를 빼서, 배경보다 어두운 미세한 구조를 강조합니다. 이 두 연산은 불균일한 조명 조건에서 미세 결함을 검출하는 데 특히 유용합니다.


  Kernel Shape 속성으로 구조 요소의 형태를 선택할 수 있습니다. **Rect(사각형)**은 가장 일반적인 형태로 수평/수직 방향에 균일한 효과를 줍니다. **Cross(십자형)**은 수평과 수직 방향으로만 연산이 적용되어 대각선 방향의 구조를 보존합니다. **Ellipse(타원형)**은 등방성(isotropic) 연산으로 모든 방향에 균일한 효과를 제공하며, 원형 객체 처리에 적합합니다.


**응용 분야**


    - 노이즈 제거 (Open): 이진화 후 발생하는 미세 노이즈를 제거하면서 객체 크기 보존

    - 구멍 메우기 (Close): 객체 내부의 불완전한 영역을 메우면서 원래 크기 유지

    - 윤곽선 추출 (Gradient): 팽창-침식 차이로 객체의 정밀한 외곽선 획득

    - 미세 결함 검출 (TopHat/BlackHat): 불균일 조명 환경에서 밝거나 어두운 미세 결함 강조

    - 텍스처 분석: 다양한 커널 형태와 연산 조합으로 표면 텍스처 특징 추출


### 13.3 Threshold 노드

  이미지 이진화(Thresholding)는 그레이스케일 이미지의 각 픽셀을 임계값과 비교하여 이진 이미지(흑백)로 변환하는 가장 기본적인 영상 분할(segmentation) 기법입니다. 적절한 임계값 선정은 이진화의 품질을 결정하는 핵심 요소이며, MVXTester는 고정 임계값, 적응형 임계값, 자동 임계값(Otsu) 세 가지 방식을 제공합니다.


#### 13.3.1 Threshold 노드

  고정 임계값을 사용하여 이미지를 이진화하는 기본 임계값 노드입니다. 사용자가 지정한 단일 임계값을 기준으로 모든 픽셀을 분류하며, Binary, BinaryInv, Trunc, ToZero, ToZeroInv 등 다양한 임계값 타입을 지원합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 이진화할 입력 이미지 |
| Output | Result | Mat | 이진화된 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Threshold | double | 128.0 | 0.0 ~ 255.0 | 이진화 기준 임계값 |
| Max Value | double | 255.0 | 0.0 ~ 255.0 | 임계값 이상 픽셀에 할당할 최대값 (THRESH_BINARY 용) |
| Type | ThresholdTypes | Binary | Binary, BinaryInv, Trunc, ToZero, ToZeroInv | 임계값 처리 방식 |


**기능 설명**

  Threshold 노드는 OpenCV의 `Cv2.Threshold` 함수를 사용하여 고정 임계값 기반의 이진화를 수행합니다. **Binary** 타입에서는 픽셀 값이 임계값보다 크면 Max Value를, 그렇지 않으면 0을 할당합니다. **BinaryInv**는 Binary의 반전으로, 임계값 이하에 Max Value를 할당합니다. **Trunc**는 임계값보다 큰 픽셀만 임계값으로 잘라냅니다.


  **ToZero** 타입은 임계값보다 큰 픽셀은 원래 값을 유지하고 나머지는 0으로 설정합니다. **ToZeroInv**는 그 반대로, 임계값 이하의 픽셀만 원래 값을 유지합니다. 이러한 다양한 타입을 통해 단순 이진화 외에도 부분적인 값 보존이 가능한 유연한 임계값 처리를 수행할 수 있습니다.


  고정 임계값은 조명 조건이 일정한 환경에서 빠르고 효과적인 이진화 방법입니다. 그러나 이미지 전체에 동일한 임계값을 적용하므로, 조명이 불균일하거나 배경 밝기가 위치에 따라 달라지는 경우에는 Adaptive Threshold나 Otsu Threshold 노드를 사용하는 것이 더 적합합니다.


**응용 분야**


    - 단순 이진화: 균일 조명 환경에서 객체와 배경을 빠르게 분리

    - 마스크 생성: 특정 밝기 범위의 영역을 선택하는 마스크 이미지 생성

    - 문서 스캔: 고르게 조명된 문서 이미지에서 텍스트와 배경 분리

    - 밝기 필터링: Trunc/ToZero 타입으로 특정 밝기 이상/이하만 선택적 처리


#### 13.3.2 Adaptive Threshold 노드

  적응형 임계값을 사용하여 이미지를 이진화하는 노드입니다. 이미지의 각 픽셀 주변 영역의 평균 또는 가우시안 가중 평균을 기준으로 국소적인 임계값을 자동으로 계산하므로, 조명이 불균일한 환경에서도 효과적인 이진화가 가능합니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 이진화할 입력 이미지 (다채널인 경우 자동 그레이스케일 변환) |
| Output | Result | Mat | 적응형 이진화된 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Max Value | double | 255.0 | 0.0 ~ 255.0 | 임계값 조건 만족 시 할당할 최대값 |
| Method | AdaptiveThresholdTypes | MeanC | MeanC, GaussianC | 적응형 임계값 계산 방법 |
| Type | ThresholdTypes | Binary | Binary, BinaryInv | 임계값 처리 방식 |
| Block Size | int | 11 | 3 ~ 99 | 국소 임계값 계산에 사용할 이웃 블록 크기 (홀수) |
| C | double | 2.0 | -100.0 ~ 100.0 | 평균값에서 차감할 상수 |


**기능 설명**

  Adaptive Threshold 노드는 각 픽셀의 국소 이웃 영역을 기반으로 개별적인 임계값을 계산합니다. **MeanC** 방법은 Block Size로 지정된 이웃 영역의 단순 평균을 임계값으로 사용하고, **GaussianC** 방법은 가우시안 가중 평균을 사용합니다. 계산된 평균에서 상수 C를 차감한 값이 해당 픽셀의 최종 임계값이 됩니다. C 값이 양수이면 임계값이 낮아져 더 많은 픽셀이 흰색으로 분류되고, 음수이면 그 반대입니다.


  입력 이미지가 컬러(다채널)인 경우, 내부적으로 `Cv2.CvtColor(BGR2GRAY)`를 자동으로 적용하여 그레이스케일로 변환합니다. Block Size는 반드시 홀수여야 하며, 짝수가 입력되면 자동으로 +1 보정됩니다. 또한 최소값은 3으로 제한됩니다.


  적응형 임계값은 조명이 불균일하거나 그라데이션이 있는 이미지에서 특히 효과적입니다. 예를 들어 자연광 아래에서 촬영된 문서 이미지나, 조명 각도에 따라 밝기가 달라지는 금속 표면 검사 등에서 고정 임계값보다 우수한 이진화 결과를 제공합니다. Block Size가 클수록 더 넓은 영역의 밝기 변화를 고려하며, 작을수록 국소적 디테일에 민감하게 반응합니다.


**응용 분야**


    - 불균일 조명 이진화: 조명 조건이 이미지 영역별로 다른 환경에서의 안정적 이진화

    - 문서 OCR 전처리: 그림자나 조명 그라데이션이 있는 문서 스캔 이미지의 텍스트 추출

    - 곡면 제품 검사: 곡면에 의한 조명 반사 차이가 있는 제품의 표면 결함 검출

    - 현미경 이미지 분석: 불균일한 배경 밝기를 가진 현미경 이미지에서 세포/입자 분리

    - 반투명 재질 검사: 두께에 따라 투과광 밝기가 달라지는 재질의 결함 검출


#### 13.3.3 Otsu Threshold 노드

  Otsu의 알고리즘을 사용하여 최적 임계값을 자동으로 결정하는 이진화 노드입니다. 이미지의 히스토그램을 분석하여 전경과 배경 사이의 클래스 간 분산을 최대화하는 임계값을 자동으로 계산하므로, 사용자가 임계값을 수동으로 설정할 필요가 없습니다.


**포트 상세**


| 방향 | 이름 | 타입 | 설명 |
| --- | --- | --- | --- |
| Input | Image | Mat | 이진화할 입력 이미지 (다채널인 경우 자동 그레이스케일 변환) |
| Output | Result | Mat | Otsu 자동 이진화된 결과 이미지 |


**속성**


| 이름 | 타입 | 기본값 | 범위 | 설명 |
| --- | --- | --- | --- | --- |
| Max Value | double | 255.0 | 0.0 ~ 255.0 | 임계값 이상 픽셀에 할당할 최대값 |


**기능 설명**

  Otsu Threshold 노드는 Otsu의 이진화 알고리즘을 사용하여 최적의 전역 임계값을 자동으로 결정합니다. 이 알고리즘은 이미지의 픽셀 값 히스토그램을 분석하여, 두 클래스(전경과 배경)의 클래스 간 분산(inter-class variance)을 최대화하는 임계값을 탐색합니다. 이렇게 계산된 임계값은 전경과 배경을 가장 잘 분리하는 최적값으로, 수동 조정 없이도 우수한 이진화 결과를 제공합니다.


  내부적으로 `Cv2.Threshold`를 `ThresholdTypes.Binary | ThresholdTypes.Otsu` 플래그 조합으로 호출합니다. 이 경우 사용자가 지정한 임계값(thresh=0)은 무시되고, Otsu 알고리즘이 자동으로 계산한 임계값이 적용됩니다. 입력 이미지가 컬러인 경우, 내부적으로 그레이스케일 변환을 자동 수행합니다.


  Otsu 알고리즘은 이미지의 히스토그램이 이봉(bimodal) 분포를 가질 때 가장 효과적으로 동작합니다. 즉, 전경과 배경이 명확한 두 개의 밝기 그룹으로 나뉘는 이미지에서 최적의 결과를 제공합니다. 반면, 복잡한 밝기 분포를 가지거나 조명이 불균일한 이미지에서는 Adaptive Threshold가 더 적합할 수 있습니다. 속성이 Max Value 하나뿐이므로 설정이 매우 간단하며, 빠른 프로토타이핑이나 자동화된 파이프라인에서 유용합니다.


**응용 분야**


    - 자동 이진화: 임계값을 수동으로 결정하기 어려운 환경에서 자동 최적 이진화

    - 배치 처리: 다양한 밝기의 이미지를 일괄 처리할 때 개별 임계값 조정 없이 자동 적응

    - 프로토타이핑: 빠른 알고리즘 검증 시 임계값 튜닝 없이 즉시 이진화 결과 확인

    - 이봉 분포 분리: 전경과 배경이 명확히 구분되는 산업 검사 이미지의 자동 분할

    - 품질 검사 자동화: 생산 라인에서 제품 밝기 변동에 자동으로 적응하는 검사 파이프라인 구축
