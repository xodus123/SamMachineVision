# MVXTester 트러블슈팅 가이드

## 오류 목록 요약

자주 발생하는 오류와 해결 방법:
- 프로그램이 시작되지 않음: .NET 8 런타임 확인, 관리자 권한 실행
- 프로젝트 파일 열기/저장 실패: 파일 형식(.mvxp/.mvx), 쓰기 권한 확인
- 노드 실행 시 오류: 필수 입력 포트 연결, 이미지 형식 확인
- "입력 이미지가 없습니다": Image Read 파일 경로, 카메라 연결 확인
- "포트 타입이 일치하지 않습니다": Mat→Mat, 숫자→숫자 등 타입 호환성
- 실행 결과가 안 나옴: Execute(F5) 실행 여부, Image Show 연결 확인
- 실행이 느림: 이미지 Resize, 불필요 노드 삭제, 루프 횟수 조정
- 카메라 안 뜸: 물리적 연결, 드라이버, SDK 설치 확인
- 카메라 이미지 검은색: 렌즈 캡, 노출 설정, 해상도 조정
- Crop 결과 검은 화면: ROI 좌표가 이미지 범위 내인지 확인
- Threshold 결과 이상: 그레이스케일 변환 여부, 임계값 범위 확인
- 윤곽선 미검출: 이진화 선행 처리, 임계값 조정
- 에지 검출 과다/부족: Threshold1/Threshold2 조정, 블러 전처리
- 색상 검출 안 됨: HSV 색공간 변환, InRange 범위 조정
- 템플릿 매칭 정확도 낮음: 템플릿 크기, 매칭 방법 변경
- 메모리 부족: Resize로 이미지 축소, Stream 모드 주의
- 챗봇 답변 품질 낮음: 더 큰 모델 또는 API 모델 사용

## 일반 오류

### 프로그램이 시작되지 않음
1. .NET 8 런타임이 설치되어 있는지 확인합니다.
2. 관리자 권한으로 실행해 봅니다.
3. 바이러스 백신이 차단하고 있지 않은지 확인합니다.

### 프로젝트 파일 열기 실패
1. .mvxp 또는 .mvx 파일인지 확인합니다.
2. 파일이 손상되지 않았는지 확인합니다. .mvxp는 ZIP 형식이므로 압축 해제가 가능해야 합니다.
3. 최신 버전의 MVXTester에서 저장된 파일인지 확인합니다. 이전 버전 파일은 호환되지 않을 수 있습니다.

### 프로젝트 저장 실패
1. 저장 경로에 쓰기 권한이 있는지 확인합니다.
2. 파일명에 특수문자가 포함되어 있지 않은지 확인합니다.
3. 디스크 용량이 충분한지 확인합니다.

## 실행 관련 오류

### 노드 실행 시 오류 발생
1. 모든 필수 입력 포트가 연결되어 있는지 확인합니다.
2. 입력 이미지가 올바른 형식인지 확인합니다. 일부 노드는 그레이스케일만, 일부는 컬러만 지원합니다.
3. 속성값이 유효한 범위 내인지 확인합니다.

### "입력 이미지가 없습니다" 오류
1. Image Read 노드의 파일 경로가 올바른지 확인합니다.
2. 이미지 파일이 실제로 존재하는지 확인합니다.
3. 카메라 노드를 사용하는 경우, 카메라가 연결되어 있고 정상 작동하는지 확인합니다.
4. 상위 노드가 정상적으로 실행되었는지 확인합니다.

### "포트 타입이 일치하지 않습니다" 오류
1. 연결하려는 두 포트의 데이터 타입이 호환되는지 확인합니다.
2. Mat 출력은 Mat 입력에, 숫자 출력은 숫자 입력에 연결해야 합니다.
3. 타입 변환이 필요하면 적절한 변환 노드를 사용합니다.

### 실행 결과가 나오지 않음
1. Execute 또는 Force Execute 버튼을 클릭했는지 확인합니다. F5로 전체 실행, F6으로 선택 노드만 실행할 수 있습니다.
2. Image Show 노드가 연결되어 있는지 확인합니다.
3. 노드 순서가 올바른지 확인합니다. 데이터는 왼쪽에서 오른쪽으로 흐릅니다.

### 실행이 느림
1. 이미지 해상도가 너무 높으면 Resize 노드로 크기를 줄여봅니다.
2. 불필요한 노드가 없는지 확인합니다.
3. For/While 루프의 반복 횟수가 적절한지 확인합니다.
4. 카메라 입력의 경우, 해상도와 프레임 레이트를 조절해봅니다.

## 카메라 관련 오류

### 카메라가 목록에 나타나지 않음
1. 카메라가 PC에 물리적으로 연결되어 있는지 확인합니다.
2. 카메라 드라이버가 설치되어 있는지 확인합니다.
3. Camera 노드의 속성에서 새로고침 버튼을 클릭합니다.
4. 다른 프로그램이 카메라를 사용하고 있지 않은지 확인합니다.

### 카메라 이미지가 검은색
1. 카메라 렌즈 캡이 열려 있는지 확인합니다.
2. 노출 시간(Exposure)이 적절한지 확인합니다.

3. 카메라 게인(Gain) 값을 높여봅니다.

### HIKROBOT/Cognex 카메라 연결 실패
1. 해당 카메라의 SDK가 설치되어 있는지 확인합니다.
2. 네트워크 설정(IP 대역)이 올바른지 확인합니다.
3. 방화벽이 카메라 통신을 차단하고 있지 않은지 확인합니다.

## 이미지 처리 오류

### Crop 결과가 검은 화면으로 나옴
1. Crop 노드의 X, Y, Width, Height 속성값을 확인합니다. 값이 0이면 빈 영역을 자르게 되어 검은 화면이 나옵니다.
2. X, Y 좌표가 원본 이미지 범위 안에 있는지 확인합니다. 이미지 크기보다 큰 좌표를 지정하면 검은 화면이 됩니다.
3. Width, Height가 0보다 큰 값인지 확인합니다.
4. 속성 편집기의 미리보기 이미지에서 마우스로 드래그하면 ROI 영역을 직접 지정할 수 있습니다.
5. 미리보기 이미지를 더블클릭하면 전체 이미지 크기로 초기화됩니다.

### Threshold 결과가 이상함
1. 입력 이미지가 그레이스케일인지 확인합니다. 컬러 이미지는 먼저 Convert Color로 변환합니다.
2. 임계값을 조절해봅니다. Otsu Threshold는 자동으로 최적값을 찾습니다.
3. 조명이 불균일하면 Adaptive Threshold를 사용합니다.

### 윤곽선이 검출되지 않음
1. 입력이 이진(binary) 이미지인지 확인합니다. Find Contours는 이진 이미지가 필요합니다.
2. Threshold나 Canny Edge 노드를 먼저 적용합니다.
3. 객체가 너무 작거나 크면 Contour Filter로 면적 범위를 조정합니다.

### 에지 검출 결과가 너무 많음/적음
1. Canny의 Threshold1, Threshold2 값을 조정합니다. 값이 높을수록 에지가 적게 검출됩니다.
2. 전처리로 Gaussian Blur를 적용하여 노이즈를 줄입니다.
3. 이미지 콘트라스트가 충분한지 확인합니다.

### 색상 검출이 안됨 (InRange)
1. 입력 이미지를 HSV로 변환했는지 확인합니다. InRange는 보통 HSV 공간에서 사용합니다.
2. HSV 범위를 조정합니다. H값은 0~180, S와 V값은 0~255입니다.
3. 조명 조건에 따라 범위를 넓게 설정해야 할 수 있습니다.

### 템플릿 매칭 정확도가 낮음
1. 템플릿 이미지가 대상과 동일한 스케일인지 확인합니다.
2. 매칭 방법(Method)을 변경해봅니다. TM_CCOEFF_NORMED가 일반적으로 좋습니다.
3. 입력 이미지와 템플릿의 채널 수가 같은지 확인합니다.

## 스크립트 노드 오류

### Python Script 실행 실패
1. 시스템에 Python이 설치되어 있는지 확인합니다.
2. 명령 프롬프트에서 python 또는 python3 명령이 작동하는지 확인합니다.
3. 스크립트에서 필요한 패키지(numpy, cv2 등)가 설치되어 있는지 확인합니다.
4. 스크립트는 sys.argv[1]로 입력 이미지를 읽고 sys.argv[2]로 결과를 저장해야 합니다.

### C# Script 실행 실패
1. 스크립트 코드에 문법 오류가 없는지 확인합니다.
2. OpenCvSharp 네임스페이스가 올바르게 사용되었는지 확인합니다.
3. 입력 이미지를 inputImage 변수로 받고 result 변수에 결과를 할당해야 합니다.

## 통신 노드 오류

### Serial Port 연결 실패
1. COM 포트 번호가 올바른지 장치 관리자에서 확인합니다.
2. Baud Rate 등 통신 설정이 대상 장치와 일치하는지 확인합니다.
3. 다른 프로그램이 같은 포트를 사용하고 있지 않은지 확인합니다.

### TCP 연결 실패
1. IP 주소와 포트 번호가 올바른지 확인합니다.
2. 방화벽이 해당 포트를 차단하고 있지 않은지 확인합니다.
3. 대상 서버/클라이언트가 실행 중인지 확인합니다.

## AI/ML 노드 오류

### MediaPipe 노드 실행 실패
1. 모델 파일(.tflite)이 Models 폴더에 있는지 확인합니다.
2. 입력 이미지가 컬러(BGR)인지 확인합니다.
3. 이미지 크기가 너무 크면 Resize로 줄여봅니다.

### YOLO 노드 실행 실패
1. ONNX 모델 파일이 올바른 경로에 있는지 확인합니다.
2. 모델이 YOLOv8 형식인지 확인합니다.
3. 입력 이미지가 유효한지 확인합니다.

### OCR 텍스트 인식이 안됨
1. 이미지에 충분한 해상도의 텍스트가 있는지 확인합니다.
2. 텍스트가 정방향인지 확인합니다. 회전된 텍스트는 먼저 Rotate로 보정합니다.
3. Tesseract OCR은 인쇄된 문서 텍스트에 최적화되어 있습니다. 로고나 디자인 텍스트는 인식이 어렵습니다.
4. PaddleOCR은 다국어와 다양한 텍스트 스타일을 더 잘 지원합니다.

### LLM/VLM 노드 API 오류
1. API 키가 올바르게 설정되어 있는지 확인합니다.
2. 인터넷 연결 상태를 확인합니다.
3. API 사용량 한도를 초과하지 않았는지 확인합니다.

## 챗봇 관련 오류

### 챗봇이 응답하지 않음
1. Ollama 모드: Ollama가 실행 중인지 확인합니다. 작업 관리자에서 ollama 프로세스를 확인합니다.
2. API 모드: API 키가 올바른지, 인터넷이 연결되어 있는지 확인합니다.
3. 챗봇 설정에서 모델이 올바르게 선택되어 있는지 확인합니다.

### Ollama 서버 연결 실패
1. Ollama가 설치되어 있는지 확인합니다.
2. 터미널에서 ollama serve 명령으로 수동 시작합니다.
3. 챗봇 설정에서 설정된 Ollama 서버 주소로 브라우저에서 접속하여 서버 상태를 확인합니다.
4. 방화벽이 Ollama 서버 포트를 차단하고 있지 않은지 확인합니다.
5. 챗봇 설정에서 Ollama 서버 주소가 올바른지 확인합니다.

### 챗봇 답변 품질이 낮음
1. 더 큰 모델(qwen3.5:9b 등)로 변경해봅니다.
2. API 모델(Claude, GPT, Gemini)을 사용하면 더 높은 품질의 답변을 받을 수 있습니다.
3. 질문을 구체적으로 작성합니다. "에지 검출 어떻게 해?" 보다 "Canny Edge 노드의 Threshold 값을 어떻게 설정하나요?"가 더 좋은 답변을 받습니다.

### API 모드 설정 방법
1. 챗봇 설정에서 제공자를 OpenAI, Claude, Gemini 중 선택합니다.
2. 해당 서비스의 API 키를 입력합니다. API 키는 각 서비스의 공식 사이트에서 발급받을 수 있습니다.
3. 원하는 모델을 선택하고 확인을 클릭합니다.
4. API 모드는 인터넷 연결이 필수이며, 사용량에 따라 비용이 발생할 수 있습니다.
5. 모바일 환경에서는 로컬 모델(Ollama)을 사용할 수 없으므로 API 모드를 사용해야 합니다.

### API 키 관련 오류
1. API 키가 올바르게 입력되었는지 확인합니다. 앞뒤 공백이 포함되지 않았는지 확인합니다.
2. API 키가 만료되거나 비활성화되지 않았는지 해당 서비스 사이트에서 확인합니다.
3. API 사용량 한도를 초과하면 오류가 발생합니다. 서비스 대시보드에서 사용량을 확인합니다.
4. 무료 플랜에서는 요청 횟수 제한이 있을 수 있습니다.

## 단축키 참고

### 자주 사용하는 단축키
1. F5: 전체 실행 (Execute All)
2. F6: 선택 노드 실행 (Execute Selected)
3. Ctrl+S: 프로젝트 저장
4. Ctrl+Z: 실행 취소 (Undo)
5. Ctrl+Y: 다시 실행 (Redo)
6. Ctrl+C/V: 노드 복사/붙여넣기
7. Delete: 선택 노드 삭제
8. Space: 캔버스 패닝 모드
9. Ctrl+마우스휠: 캔버스 확대/축소

## 성능 최적화 팁

### 메모리 사용량 줄이기
1. 처리할 필요 없는 대용량 이미지는 Resize로 줄입니다.
2. 사용하지 않는 노드는 삭제합니다.
3. Stream 모드 대신 단일 실행 모드를 사용합니다.

### 처리 속도 높이기
1. 전처리 단계에서 이미지를 Crop하여 관심 영역만 처리합니다.
2. 불필요한 중간 표시(Image Show)를 줄입니다.
3. 적절한 이미지 해상도를 선택합니다. 대부분의 검사는 640x480 이상이면 충분합니다.
